# ❗ `미션 설계`
- ### `무엇을 어떻게 왜 개발할 것인지 정리하고 계획한다.`
- 기능을 구현하기 전에 기능 목록을 만들고,  
  `실행 흐름 정리` ➡️ `기능 목록 정리` ➡️ `기능 명세 작성` ➡️ `구현`
- 기능 단위로 커밋하는 방식으로 진행한다.  
  명세에 맞춰 순차적으로 기능을 구현하고 커밋한다
- **`기능 요구 사항에 기재되지 않은 내용은 스스로 판단하여 구현한다.`**
  1. 명시된 요구 사항을 준수한다.
  2. 비명시 요구 사항의 구현 범위를 판단한다. 
     1. 사이클2에서 다룰 부분 말고 전부를 구현할 것인가?
     2. 명시된 요구 사항과 유효한 실행 흐름 정도만 구현할 것인가? 
- 개발할 기능에 대한 설계 뿐만 아니라, 개발 과정에 대한 설계도 진행한다.
    - 어느 수준까지 구현할 것인가? 
      - 명시된 요구 요구 사항을 준수한다면 
      - 구현 원칙을 준수해 
      - 피드백을 먼저 받고 기능을 추가 구현한다
    - 그 이유는?
      - 구현의 목표는 완벽한 코드가 아닌 피드백을 받을 수 있는 코드다
      - 부족한 부분을 인지하고 공유하고 개선한다 
      - 세부 구현보단 방향성과 의도를 드러내는 것이 중요하다
    - 어떤 일정으로 구현할 것인가? 
      - 데드라인 : 목요일 오후 18시 PR
    - 그 이유는?
      - 이전 기수와 동등한 목표 달성
      - 더 빠른 더 많은 피드백 
      - > `리뷰어(로빈) 코멘트`  
        리뷰 일정을 알려달라는 이유는 제 일정을 맞추려는 것도 있지만,  
        리뷰이 여러분들의 일정 산출 및 관리를 잘 할 수 있도록 도와주려는 목적이 더 큽니다.   
        산출이 어렵더라도 메타인지를 하기 위한 데이터가 되는 경험일테니   
        대략적으로라도 말해주면 좋을거 같아용.
    - 무엇을 목표로 구현할 것인가? 
      - 함께 고민한 내용을 드러낼 수 있는 코드
      - 이전 미션에서 배운 내용을 반영한 코드
      - 지금의 요구사항만이 아닌, 이후의 확장에 대비된 코드
    - 그 이유는?
      - 동작하는 코드도 물론 피드백을 받을 수 있는 코드지만, 
      - 그 동작을 통해 개발자의 의도를 표현할 수 있어야 한다 생각.
      - `돌아가는 쓰레기` + `그런데 제작자의 의도가 보이는`

# 🛠️ 실행 흐름

```java
1. 선수(초), 후수(한) 플레이어의 이름을 입력받는다.
↓
2. 장기판과 기물을 초기화하고 게임을 시작한다.
↓
┌─→ 3. 현재 턴인 플레이어에게 이동할 기물과 도착 위치를 입력받는다.
│            ↓
│   4. 해당 이동이 규칙(공통/개별)에 맞는지 검증한다.
│   ⬆️ ↙ [불가능]                       ↘ [가능]
│  (재입력)                     5. 기물을 이동하고 장기판을 갱신한다.
│  
------------------------1차 구현 및 PR 포인트---------------------------
│                                         
│                                         ↓
│                               6. 상대의 '장'이 잡혔는지 확인한다.
│                                 ↙ [아니오]          ↘ [예]
└─────────────────────────────────┘             7. 게임을 종료한다.
                                                      ↓
                                                8. 승패 결과를 출력한다.
```

# 📄 기능 명세

- [ ] 1. [IO] 선수(초)와 후수(한) 플레이어의 이름을 입력받는다.  `class InputView`

  - [ ]  [출력] 초 진영의 플레이어 이름 입력 안내문 출력 - `class OutputView`
  - [ ]  [입력] 초 진영의 플레이어 이름 입력 - `public String readPlayerName`
  - [ ]  [출력] 한 진영의 플레이어 이름 입력 안내문 출력 - `class OutputView`
  - [ ]  [입력] 한 진영의 플레이어 이름 입력 - `public String readPlayerName`
  - [ ] [예외 처리] 입력이 비어있는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.


- [ ]  [중간 과정] 플레이어를 관리할 일급 컬렉션을 생성한다. `class Players`  -
    - [ ]  [예외 처리] 플레이어 닉네임이 중복되는 경우, `IllegalArgumentException` 을 발생시킨다.
      `private void validateDuplicatedName(String choPlayerName, String hanPlayerName)`
    - [ ]  [규칙] 각 진영의 플레이어 이름으로 플레이어 일급 컬렉션 생성
      `public Players(String choPlayer, String hanPlayer)`
    - [ ]  [규칙] 초 진영의 플레이어 이름으로 초 진영의 플레이어 생성
      `public Player(String name)`
    - [ ]  [규칙] 한 진영의 플레이어 이름으로 초 진영의 플레이어 생성
      `public Player(String name)`

- [ ] 2. [출력] 현재 장기판의 상태를 출력한다. `class OutputView`

  - [ ]  [출력] 현재 장기판의 상태를 출력한다
    `public void printBoardStatus(BoardDTO boardDto)`
  - [ ]  [규칙] 장기판 `class Board` 의
    기물 배치 상태 `Map<Position, Piece> piecePosition` 를
    장기판 DTO `class BoardDTO` 가 받아 DTO 생성
    `public BoardDTO from(Board board)`
  - [ ]  [출력] 장기판의 기물 배치 상태에 맞춰 장기판의 상태를 출력한다
    `public void printBoardStatus(BoardDTO boardDto)`

- [ ] 3. [중간 과정] 장기판과 전체 기물을 올바른 초기 위치에 배치한다.

  - //[ ] [규칙] 한이 먼저 상과 마의 위치를 배치한다.
  - // [ ] [규칙] 이후, 초가 상과 마의 위치를 배치한다.

4. [출력] 현재 장기판의 상태를 출력한다
   `public void printBoardStatus(BoardDTO boardDto)`

[ ] 5. [입력] 현재 턴의 플레이어에게 이동할 기물을 입력받는다.

- [ ] [규칙] 초가 먼저 시작한다.
- [ ] [예외 처리]

[ ] 6. [중간 과정] 공통 이동 규칙을 검증한다.

- [ ] [규칙] 도착 위치에 같은 팀 기물이 있으면 이동할 수 없다.
- [ ] [규칙] 이동 이후 자신의 '장'이 위험한 상태(장군)가 된다면, 그 이동은 허용되지 않는다.

[ ] 7. [중간 과정] 개별 기물의 이동 규칙을 검증한다.

- [ ] [규칙] 장: 궁성 내부에서만 한 칸 이동할 수 있고, 궁성 내부 대각선 이동도 가능하다.
- [ ] [규칙] 차: 상하좌우로 거리 제한 없이 이동할 수 있다. 단, 이동 경로 중간에 기물이 있으면 뛰어넘을 수 없다.
- [ ] [규칙] 포: 이동 경로 사이에 반드시 다른 기물이 하나 이상 있어야 하며, 그 기물이 '포'이면 안 된다.
- [ ] [규칙] 포: 도착 위치의 기물이 '포'인 경우에는 잡을 수 없다.
- [ ] [규칙] 마: 직진 한 칸 후 대각선 한 칸 이동한다. 직진하는 첫 칸에 기물이 있으면 이동할 수 없다.
- [ ] [규칙] 상: 직진 한 칸 후 대각선으로 두 칸 이동한다. 이동 경로에 있는 칸이 막혀 있으면 이동할 수 없다.
- [ ] [규칙] 사: 궁성 내부에서만 한 칸 이동할 수 있고, 궁성 내부 대각선 이동도 가능하다.
- [ ] [규칙] 졸/병: 한 칸씩 이동할 수 있으며, 전진 및 좌우 이동만 가능하다. (후퇴 불가 등 지정된 방향 조건 적용)

[ ] 8. [중간 과정] 해당 기물이 이동할 수 있는 위치를 출력한다.

[ ] 9. [입력]  도착 위치를 입력받는다.

[ ] 10. [중간 과정] 검증을 통과하면 기물을 이동시키고, 턴을 교체한다.

[ ] 11. [중간 과정] 턴을 반복한다.

[ ] 12. [중간 과정] '장'이 잡혔는지 판단하여 게임 종료 여부를 결정한다.

- [ ] [규칙] 한쪽의 '장'이 잡히면 즉시 게임이 끝난다.

[ ] 13. [출력] 최종 승패 결과를 출력한다.
