# 당연한 원칙, 직접 검증하기

> 당연하게 받아들였던 개발 원칙이나 철학을 직접 실험을 통해 확인하는 미션입니다.  
> 가설을 세우고 코드를 구현해 데이터를 수집한 뒤, 실험 보고서로 결과를 정리하며  
> 증거와 논리에 기반한 기술적 의사결정 역량을 기릅니다.

## 🎯 개요

이 미션은 "남들이 좋다고 하니까"가 아니라, **정말 그런지 직접 확인해보는** 경험을 하는 미션입니다.

개발 세계에는 당연하게 받아들여지는 원칙과 격언이 많습니다. "함수는 짧을수록 좋다", "DRY 원칙을 지켜라" 같은 것들입니다.

이 미션에서는 그중 하나를 골라 가설을 세우고, 코드를 작성하고, 직접 확인하여 정말 그런지를 판단합니다. 그 과정과 결과를 실험 보고서 형식으로 정리하며, 증거와 논리로 기술적 의사결정을 주장하는 경험을 합니다.

여기서의 실험은 학술 논문 수준의 엄밀한 실험이 아닙니다. **"이 원칙이 정말 그런지 내가 직접 확인해보려면 뭘 해봐야 할까?"**를 고민하고, 실제로 해보는 것입니다. 완벽한 실험 설계보다 **"의심 → 확인 시도 → 나름의 결론"**이라는 사고 흐름을 경험하는 것이 핵심입니다. 가설이 틀린 것으로 나와도 괜찮습니다. "이 원칙이 항상 맞는 건 아니구나"라는 발견 자체가 이 미션의 목표입니다.

## 🚀 학습 목표

* 남들이 좋다고 해서 무비판적으로 수용하는 태도를 버린다.
* 추상적인 개념(좋은 코드, 가독성 등)을 구체적인 지표로 치환하여 설계하는 능력을 기른다.
* 증거와 논리로 기술적 의사결정을 주장한다.
* 소프트웨어에 `은탄환(Silver Bullet)`은 없다는 것을 체감한다.

## 🏃‍♂️ 진행 방식

이 미션은 4주에 걸쳐 3단계로 진행됩니다. 2~3주차는 필수 미션을 수행하면서 병행합니다.

* **1단계 실험 계획** (레벨1 4주차): 확인할 개발 원칙을 선택하고, 가설을 세우고, 어떻게 확인할지 계획합니다.
    * 기존에 당연하게 받아들였던 개발 원칙/철학/명언 중 확인할 주제 선정
    * 선택한 원칙을 구체적인 가설로 변환
    * 비교 대상 설정, 확인 지표 정의, 실험 절차 수립
* **2단계 실행** (레벨1 5~6주차): 실험 계획에 따라 실제 코드를 작성하고 결과를 확인합니다.
    * 원칙 미적용 버전과 적용 버전 구현
    * 정한 지표에 따라 결과 확인 및 기록
    * 실험 과정에서 발생한 이슈, 예상 외 상황 기록
* **3단계 분석 및 마무리** (레벨1 7주차): 확인한 결과를 분석하고 실험 보고서 형식으로 정리합니다.
