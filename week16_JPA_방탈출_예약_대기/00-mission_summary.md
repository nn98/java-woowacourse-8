# 소개

> JdbcTemplate > JPA 이관, 예약 대기 기능을 새로 얹기.  
> 영속성 컨텍스트가 만든 자동화의 비용을 직접 확인.

## 개요

어노테이션이 만드는 실제 SQL 추적하기.  
JPA는 많은 것을 자동화한다. 이는 곧 많은 것을 은닉한다는 뜻이며  
내부 로직을 모르면 의도와 다른 SQL이 나가도 인지하지 못함.

## 목표

- JdbcTemplate/JPA 표현력 차이 본인 코드 위에서 비교
- 영속성 컨텍스트 동작 관찰 / 발행 SQL 추적
    - 1차 캐시
    - 더티 체킹
    - 쓰기 지연
    - 플러시
    - 페치
- 연관관계 매핑 결정 근거와 한계 인지
    - 단/양방향 매핑
    - 페치 전략
    - 캐스케이드
    - orphanRemoval
- N+1 문제 식별 / 해결법 비교 사용
- 트랜잭션 경계/동시성 의식하며 도메인 로직 위치 결정

영속성 컨텍스트 / 더티 체킹 / 쓰기 지연 / orphanRemoval / 페치 조인 / JPQL
