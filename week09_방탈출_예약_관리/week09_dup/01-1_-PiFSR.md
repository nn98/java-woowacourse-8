<h2><strong>1. Problem-Driven</strong></h2>
<p><strong>→ 목적과 문제에서 시작하라.</strong></p>
<ul>
<li>“이걸 왜 배우는가?”를 먼저 정의</li>
<li>실제 문제를 해결하는 방향으로 학습</li>
</ul>
<p>👉 예시</p>
<ul>
<li>로그인 안 됨 → 세션/토큰 학습</li>
<li>성능 느림 → 렌더링/캐싱 학습</li>
</ul>
<hr>
<h2><strong>2. i+1</strong></h2>
<p><strong>→ 문제를 쪼개고, 딱 한 단계 어려운 것만 도전하라.</strong></p>
<ul>
<li>너무 쉽지도, 너무 어렵지도 않게 (comfort zone 살짝 밖)</li>
<li>한 번에 다 이해하려 하지 말고, 한 단계씩</li>
</ul>
<p>👉 예시</p>
<ul>
<li>SOLID 전체가 아니라, 한 원칙만 먼저 적용</li>
<li>클린코드 규칙 한번에 모두 적용하지 않고, 하나씩 나눠서 적용</li>
</ul>
<hr>
<h2><strong>3. Feedback Loop</strong></h2>
<p><strong>→ 바로 해보고, 실패하고, 피드백을 반복하라.</strong></p>
<ul>
<li>Do → Fail → Feedback → Iterate</li>
<li>Output 먼저 내라 (Output First)</li>
<li>입력만 하고 출력을 통한 실패/피드백이 없다면 학습이 아니다</li>
</ul>
<p>👉 예시</p>
<ul>
<li>강의만 1시간 보지 말고, 보면서 직접 코드 작성</li>
<li>문제 풀고 틀린 뒤, 해설 보고 다시 풀기</li>
</ul>
<hr>
<h2><strong>4. Structural Thinking</strong></h2>
<p><strong>→ 연결하고 구조로 이해하라.</strong></p>
<ul>
<li>조각이 아니라 흐름, 관계, 역할로 본다</li>
<li>구조화하려 하면 자연스럽게 이해된다</li>
<li>마인드맵, 컨셉맵, 마크다운 heading 정리, depth 정리 등 활용</li>
</ul>
<p>👉 예시</p>
<ul>
<li><a href="http://google.com/" target="_blank">http://google.com</a> 입력 → DNS → 서버 → 응답 흐름으로 이해</li>
<li>JS vs TS 관계</li>
<li>이벤트 루프 vs 메인 스레드 관계 정리</li>
</ul>
<hr>
<h2><strong>5. Resourceful</strong></h2>
<p><strong>→ 가장 빠른 방법을 위해 모든 자원을 활용하라.</strong></p>
<ul>
<li>유튜브, 책, 사람, AI 가리지 말고 활용</li>
<li>책을 1페이지부터 끝까지 읽지 않고, 필요한 모든 자원을 활용하여 비순차적으로 학습</li>
</ul>
<p>👉 예시</p>
<ul>
<li>이미 타입스크립트를 잘하는 크루에게 학습 방향을 물어본다.</li>
<li>여러 자료를 비교하며 가장 빠르게 이해되는 방식 선택</li>
</ul>
